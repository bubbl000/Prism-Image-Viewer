# ImageViewer 版本记录

## v2.5.4 — 2026-03-16
**修复 PSB 显示 160x90 小图的问题（ZIP 压缩支持）**

- **根因**：现代 Photoshop 保存大尺寸 PSB 默认使用 ZIP 压缩（type 2/3），之前代码对 `compression > 1` 直接返回 null，导致回退到 160px 缩略图
- **修复**：`TryDecodePsdComposite` 新增 ZIP 压缩支持：
  - 整个 Image Data 剩余内容为单一 zlib 流，使用 `System.IO.Compression.ZLibStream` 解压
  - type 3（ZIP with prediction）解压后再做逐行 byte-level delta 解码
  - 现在完整支持全部 4 种压缩格式：Raw / RLE PackBits / ZIP / ZIP+预测

---

## v2.5.3 — 2026-03-16
**修复 PSD/PSB 显示极其模糊的问题**

- **根因**：之前提取的是 Image Resources 里的缩略图（Resource 0x040C），分辨率通常只有 160px 左右
- **修复**：新增 `TryDecodePsdComposite`，直接解码文件末尾 Image Data 块中的全分辨率合并图层，支持：
  - 色彩模式：RGB / 灰度 / CMYK（CMYK 自动转 RGB）
  - 位深度：8-bit / 16-bit（16-bit 取高字节，视觉等效）
  - 压缩方式：Raw(0) 和 RLE PackBits(1)；ZIP 压缩回退到缩略图
  - PSB 与 PSD 结构相同，Layer/Mask 长度字段 PSB 为 8 字节，其余一致
- **降级策略**：全分辨率解码失败时回退到内嵌 JPEG 缩略图（旧行为）
- 缩略图栏仍使用 `TryExtractPsdPreview`（60px 小图，无需全量解码）

## v2.5.2 — 2026-03-16
**修复 PSD/PSB 格式实际无法加载的问题**

- **PSD/PSB 内嵌预览提取**：新增 `TryExtractPsdPreview`，解析 PSD/PSB 文件的 Image Resources 块，定位 Resource ID `0x040C`（Thumbnail Resource），提取其中的 JPEG 数据用于显示；支持 PSD（version=1）和 PSB（version=2），文件流式读取无需整体加载到内存
- **缩略图也走同一路径**：缩略图生成时 PSD/PSB 同样先提取内嵌 JPEG，再按 60px 降采样，缩略图栏正常显示
- 若文件不含内嵌预览（Photoshop 保存时关闭了复合图像），给出明确提示

## v2.5.1 — 2026-03-16
**性能优化：缩略图并发加载 + 鸟瞰图防抖 + 缓存改 JPEG + Brush 静态复用**

- **缩略图并发加载**：`LoadThumbnailsAsync` 从串行 `await Task.Run` 改为 `SemaphoreSlim(3)` 控制并发，大文件夹缩略图加载速度明显提升
- **鸟瞰图防抖**：`ZoomBorder_LayoutUpdated` 高频触发时加 10ms `DispatcherTimer` 防抖，缩放/拖动时避免每帧重复计算坐标换算
- **缩略图缓存改 JPEG**：`ThumbnailCache.TrySave` 从 `PngBitmapEncoder` 改为 `JpegBitmapEncoder(85)`，缓存文件大小缩小 3-5 倍，读写速度显著提升
- **Brush 静态复用**：`AnimateToggle` 中每次 `new SolidColorBrush` 改为共享静态冻结 Brush，减少小对象 GC 分配

## v2.5 — 2026-03-12
**图片加载全面提速：渐进加载 + 原图缓存 + 预读取 + 硬件加速**

- **渐进式加载**：打开图片时立即显示磁盘缓存中的低分辨率缩略图，后台完成全分辨率解码后无缝替换，首帧响应时间 < 10ms
- **原图 LRU 内存缓存（ImageCache.cs）**：保留最近访问的 7 张原图 BitmapSource，翻页回上一张时零解码；切换文件夹时自动清空释放内存
- **预读取（Read-Ahead）**：加载当前图片后立即在后台预解码 +1、+2、-1 张，顺序浏览几乎无等待；文件切换时取消旧任务避免资源浪费
- **`LoadBitmapSource` 静态方法**：将 WIC 解码逻辑从 ShowImage 中提取，ShowImage / 预读取 / RAW 三条路径共享同一解码实现，消除代码重复
- **硬件加速开关**：常规设置新增「硬件加速渲染」切换，控制 `RenderOptions.ProcessRenderMode`（GPU 合成 vs 纯软件渲染），遇驱动兼容问题可一键切换

## v2.4 — 2026-03-12
**缩略图磁盘缓存 + 懒加载 + 内存管理 + 拖拽到外部程序**

- **缩略图磁盘缓存（ThumbnailCache.cs）**：首次生成后写入 `%TEMP%\ImageViewerThumbCache\`，Key = MD5(路径 + 修改时间)，再次打开同文件夹秒出缩略图
- **缓存自动清理**：关闭软件时写入 `.lastuse` 时间戳；下次启动若距上次关闭超过 30 分钟，自动删除全部缓存文件
- **懒加载顺序**：缩略图从当前图片索引向两侧交替加载，优先显示视口内项，不再从 0 号顺序加载
- **直接拖拽到外部程序**：鼠标拖动图片拖出窗口边界即触发文件 DragDrop（无需按 Ctrl）；窗口内拖动仍为平移；保留 Ctrl+拖 的原有行为
